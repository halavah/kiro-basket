# 问题描述 - 商品列表测试商品数据

**截图**: PixPin_2025-12-18_16-34-12.png
**时间**: 2025-12-18 16:34:12
**状态**: ⚠️ 待优化
**页面**: `http://localhost:5173/products` (商品管理 > 商品列表)

## 🐛 问题概述

商品列表显示了多条"测试商品"数据（ID: 21, 22, 23），这些是临时测试数据。

## 📊 显示内容

### 测试商品数据
- **ID**: 21, 22, 23
- **名称**: 测试商品
- **价格**: ¥9.99
- **单位**: ¥
- **库存**: 100
- **销量**: 0
- **状态**: 在售
- **操作**: 下架、编辑库存、删除

### 正常商品数据
- **ID**: 1-20
- 包含各种水果、蔬菜、肉蛋奶等正常商品数据

## 🔍 问题原因分析

1. **测试数据遗留**
   - 开发过程中创建的测试数据未清理
   - 数据库 seed.sql 可能包含这些测试数据

2. **数据库初始化脚本问题**
   - seed.sql 包含了测试商品的 INSERT 语句

## 📊 影响范围

- 商品列表展示不专业
- 可能影响统计数据准确性
- 用户体验不佳

## 🔧 解决方案

### 方案 1: 手动删除测试数据

在团长端商品管理页面，逐个点击"删除"按钮删除测试商品。

### 方案 2: 数据库清理（推荐）

修改 `aries/chapter02/database/mysql/seed.sql` ��件，删除或注释测试商品的 INSERT 语句：

```sql
-- 删除这些测试数据的 INSERT 语句
-- INSERT INTO products VALUES (21, ..., '测试商品', ...);
-- INSERT INTO products VALUES (22, ..., '测试商品', ...);
-- INSERT INTO products VALUES (23, ..., '测试商品', ...);
```

然后重新初始化数据库：

```bash
cd D:\software_yare\node192\server-init
npm run init
```

### 方案 3: SQL 直接删除

连接到 MySQL 数据库执行：

```sql
USE kiro_basket;
DELETE FROM products WHERE name = '测试商品';
```

## 🧪 测试步骤

1. 执行上述任一解决方案
2. 刷新商品列表页面
3. 确认测试商品已被删除
4. 验证正常商品数据完整

## 📝 相关文件

- 数据库脚本：`aries/chapter02/database/mysql/seed.sql`
- 前端页面：`client-captain/src/views/Product/ProductList.vue`
- 后端控制器：`server/src/controllers/product.controller.js`

## 💡 建议

在生产环境部署前：
1. 清理所有测试数据
2. 使用生产环境专用的数据初始化脚本
3. 区分开发环境和生产环境的数据
